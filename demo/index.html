<link rel="stylesheet" href="millenium-bug.css">

Finestra

Usa <dialog open> (o .mb-window) con un <header> come barra del titolo. Questa pagina stessa è una finestra. 

<dialog open>
  <header>
    <h1>-- Titolo finestra</h1>
    <button aria-label="Chiudi">✕</button>
  </header>
  <main>…contenuto…</main>
</dialog>

Tab

Un <nav> con link o bottoni. Il tab attivo si dichiara con aria-current (o aria-selected="true"), non con classi. La <section> immediatamente successiva diventa il pannello con il filo arancio. 

<nav>
  <a href="#">Anagrafica</a>
  <a href="#" aria-current="page">Dati pratica</a>
  <a href="#">12 Allegati</a>
</nav>
<section>…pannello…</section>

Righe di form

Ogni <p> dentro un <form> è una riga densa (flex) di label e campi. La larghezza dei campi si controlla con l'attributo size, come si faceva una volta. Il bottone lookup è <button class="mb-lookup">, le descrizioni decodificate sono <output>. Utilities: .mb-right spinge a destra, .mb-grow occupa lo spazio residuo. 

<form>
  <p>
    <label for="cod">Codice:</label>
    <input id="cod" size="6" value="00042" readonly>
    <button class="mb-lookup" aria-label="Cerca"></button>
    <output>SEGRETERIA GENERALE</output>
  </p>
</form>

Campi

<input>, <select> e <textarea> hanno il bordo incassato e il valore in monospace bold. readonly e disabled tornano grigi da soli. Checkbox e radio sono squadrati come da ordinanza. 

<input size="12" value="00042" readonly>
<select><option>In lavorazione</option></select>
<label><input type="checkbox" checked> Privacy firmata</label>

Stati ed evidenziazioni

Il cuore semantico della libreria: <mark> è il giallo warning, <ins> il verde ok, <em> l'alert rosso bold. Le date sono <time>, i valori calcolati <output>. Per colorare un campo, le varianti .mb-mark / .mb-ok / .mb-due direttamente sull'input. 

<mark>in attesa di conferma sala</mark>
<ins>Confermato</ins>
<em>Documenti privacy mancanti!!</em>
<input class="mb-due" value="17/06/2026" readonly>
<output><time datetime="2026-06-05">05/06/2026</time></output>

Bottoni e bottoniera

I <button> sono in rilievo 3D e "affondano" alla pressione. .mb-primary è la variante ciano. La bottoniera in fondo alla finestra è un <menu>: i bottoni dopo .mb-right si allineano a destra. 

<menu>
  <li><button>Cambio Sede</button></li>
  <li><button class="mb-right">Stampa Sintetica</button></li>
  <li><button disabled>OK</button></li>
  <li><button class="mb-primary">Annulla</button></li>
</menu>

Tabelle

Le <table> hanno griglia piatta, intestazioni grigie e righe che si selezionano in blu navy al passaggio del mouse. 

<table>
  <thead><tr><th>Prog.</th><th>Data</th></tr></thead>
  <tbody><tr><td>17</td><td>03/06/2026</td></tr></tbody>
</table>

Extra retro

Avanzamento a blocchi con <progress>, separatori <hr> a scanalatura, gruppi richiudibili con <details>, raggruppamenti con <fieldset>. 

<progress max="100" value="60">60%</progress>
<details open>
  <summary>Dati ultima scheda</summary>
  <fieldset><legend>Esiti</legend>…</fieldset>
</details>

Theming

Tutti i colori sono custom properties con prefisso --mb-: sovrascrivile su :root per cambiare epoca senza toccare la libreria. 

:root {
  --mb-window: #f5f4f1;     /* interno finestra quasi bianco */
  --mb-face: #d8d4cf;       /* grigio di campi e bottoni */
  --mb-tab: #1058a8;        /* tab blu */
  --mb-tab-active: #e8501e; /* tab attivo arancio */
  --mb-mark: #ffff00;       /* evidenza gialla */
  --mb-ok: #39e639;         /* evidenza verde */
  --mb-due: #b8e2e2;        /* evidenza ciano */
  --mb-alert: #e60000;      /* alert rosso */
  --mb-primary: #8ed8d8;    /* bottone primario ciano */
}
